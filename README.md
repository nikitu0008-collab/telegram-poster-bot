Telegram Auto-Poster Bot

Бот для автоматического сбора контента из различных источников (RSS-ленты, Telegram-каналы, генерация мемов) и публикации его в ваш Telegram-канал с управляемой очередью и защитой от дубликатов.
📌 Возможности

    Сбор постов из RSS-лент (извлекает заголовок, описание, изображение).

    Парсинг последних сообщений из Telegram-каналов (репост с указанием источника).

    Генерация мемов через Meme-API (с вероятностью 40%).

    Умная очередь публикаций – посты не дублируются, история сохраняется.

    Гибкие настройки интервалов сбора и публикации.

    Поддержка HTML-разметки в тексте постов.

    Работает асинхронно (Telethon + python-telegram-bot).

⚙️ Требования

    Python 3.8+

    Установленный пакетный менеджер pip

    Аккаунт Telegram (для работы Telethon требуется api_id и api_hash, получить можно на my.telegram.org)

    Бот-токен от @BotFather

    ID вашего канала (например, @my_channel)

🚀 Установка и настройка
1. Клонируйте репозиторий

git clone https://github.com/ваш_username/telegram-poster-bot.git
cd telegram-poster-bot

2. Создайте виртуальное окружение (рекомендуется)

python -m venv venv
source venv/bin/activate      # Linux/macOS
# или
venv\Scripts\activate         # Windows

3. Установите зависимости

pip install -r requirements.txt

4. Настройте переменные окружения

Скопируйте пример конфигурации и отредактируйте:
bash

cp .env.example .env

Заполните .env своими данными:
env

TELEGRAM_BOT_TOKEN=ваш_токен_бота
CHANNEL_ID=@имя_канала
TELEGRAM_API_ID=123456
TELEGRAM_API_HASH=ваш_api_hash

    Важно: никогда не коммитьте .env – он добавлен в .gitignore.

5. Настройте источники контента

Отредактируйте переменные SOURCE_CHANNELS и RSS_FEEDS в файле src/main.py:
python

SOURCE_CHANNELS = [
    "@news_channel",
    "@tech_updates",
]

RSS_FEEDS = [
    "https://example.com/rss",
    "https://another-feed.xml",
]

При желании измените интервалы (в минутах):
python

POST_INTERVAL_MINUTES = 8640   # сбор каждые 6 дней
PUBLISH_INTERVAL_MINUTES = 30  # публикация каждые 30 минут
MAX_POSTS_PER_COLLECT = 20     # максимум постов за один сбор

▶️ Запуск

python src/main.py

При первом запуске Telethon запросит подтверждение входа (потребуется номер телефона и код из Telegram). Сессия сохранится в файл user_session.session.

Бот сразу проверит очередь – если она пуста, выполнит сбор контента и опубликует первый пост, а затем перейдёт в режим ожидания по расписанию.

В процессе работы создаются файлы:

    posted_history.json – история опубликованных постов (хэш текста).

    queue.json – очередь ожидающих публикации постов.

    user_session.session – сессия Telethon.

🔧 Управление очередью и историей

    История сохраняется в posted_history.json – она не даёт опубликовать один и тот же пост дважды (сравнивается текст или подпись фото).

    Очередь находится в queue.json. Бот публикует посты последовательно, удаляя опубликованные.

    Если публикация не удалась (ошибка Telegram), пост возвращается в начало очереди.

⚠️ Возможные проблемы и решения
Проблема	Решение
Ошибка TelegramError при публикации	Проверьте токен бота, права на публикацию в канале, а также доступность URL для фото.
Бот не видит канал	Убедитесь, что бот добавлен в канал как администратор (или участник).
Ошибка импорта модулей	Убедитесь, что все зависимости установлены (pip install -r requirements.txt).
Telethon запрашивает код при каждом запуске	Проверьте файл user_session.session – он должен сохраняться после первого входа.
